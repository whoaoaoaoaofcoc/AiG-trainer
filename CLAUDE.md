# AiG Trainer — инструкции для Claude

## Что НЕЛЬЗЯ коммитить или пушить
- `data.json` — база пользователей (пароли, токены). Никогда не добавлять в git.
- `.DS_Store`, `*.log`, `.env`, `node_modules/` — системный мусор.
- Файлы с секретами или API-ключами в открытом виде.

## Перед любым git push — спросить пользователя
Не пушить автоматически. Всегда показать `git status` и `git diff --stat` и дождаться явного подтверждения.

## Структура проекта
- `server.js` — Express-сервер, авторизация, AI-роутинг (OpenRouter → Gemini → Groq)
- `public/` — фронтенд HTML-файлы
- `package.json` — зависимости
- `data.json` — файл с данными пользователей (не в git!)

## Деплой
Проект деплоится на Railway. Переменные окружения задаются там, не хардкодить в код.
